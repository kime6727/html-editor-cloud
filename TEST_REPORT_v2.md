## 四、发现的问题

### 🔴 P0 - 严重问题（必须修复）

#### 问题1: 数据库连接信息硬编码
**严重程度**: 🔴 严重  
**位置**: `deploy_package/.env`  
**问题描述**: 
- 数据库密码 `ServBay.dev` 直接写在配置文件中
- 生产环境应使用环境变量或密钥管理服务

**影响**: 安全风险，数据库凭证泄露  
**建议**: 
```php
// 使用环境变量
$pass = getenv('DB_PASS') ?: $env['DB_PASS'];
```

**状态**: ⚠️ 待修复

---

#### 问题2: 管理员密码过于简单
**严重程度**: 🔴 严重  
**位置**: `deploy_package/.env` 第18行  
**问题描述**: 
```
ADMIN_PASS=admin123456
```
使用常见弱密码，容易被暴力破解

**影响**: 运营后台可能被入侵  
**建议**: 
- 使用强密码（至少16位，包含大小写、数字、特殊字符）
- 添加登录失败次数限制
- 添加双因素认证

**状态**: ⚠️ 待修复

---

#### 问题3: 删除API缺少用户所有权强制验证
**严重程度**: 🔴 严重  
**位置**: `deploy_package/delete.php`  
**问题描述**: 
```php
// 权限验证：只有项目的创建者可以删除
if ($userId && isset($project['user_id']) && $project['user_id'] !== $userId) {
    http_response_code(403);
    echo json_encode(['status' => 'error', 'message' => 'Permission denied']);
    exit;
}
```

问题：
1. 如果 `$userId` 为空，则跳过验证
2. 任何知道project_id的人都可以删除项目
3. 没有验证HMAC签名中的用户身份

**影响**: 恶意用户可以删除他人项目  
**建议**: 
```php
// 强制验证用户身份
if (empty($userId) || $project['user_id'] !== $userId) {
    http_response_code(403);
    echo json_encode(['status' => 'error', 'message' => 'Permission denied']);
    exit;
}
```

**状态**: ⚠️ 待修复

---

#### 问题4: 发布API未强制验证用户Pro状态
**严重程度**: 🔴 严重  
**位置**: `deploy_package/publish.php`  
**问题描述**: 
```php
// 服务端验证Pro状态（不信任客户端）
$is_pro = false;
if ($user_id) {
    try {
        $db = db();
        $user = $db->queryOne("SELECT is_pro FROM users WHERE user_id = ?", [$user_id]);
        if ($user && $user['is_pro']) {
            $is_pro = true;
        }
    } catch (Exception $e) {
        error_log("User verification failed: " . $e->getMessage());
    }
}
```

问题：
1. 如果数据库查询失败，`$is_pro` 为 false，但不会阻止发布
2. 免费用户可以发布（只是强制5分钟过期）
3. 没有验证用户是否还有发布次数

**影响**: 免费用户可能绕过月度发布限制  
**建议**: 
```php
// 验证发布次数
if (!$is_pro) {
    $user = $db->queryOne(
        "SELECT publish_count FROM users WHERE user_id = ?", 
        [$user_id]
    );
    if ($user && $user['publish_count'] >= 1) {
        http_response_code(403);
        echo json_encode(['status' => 'error', 'message' => 'Monthly publish limit reached']);
        exit;
    }
}
```

**状态**: ⚠️ 待修复

### 🟡 P1 - 重要问题（建议修复）

#### 问题5: 数据存储混合使用数据库和JSON文件
**严重程度**: 🟡 重要  
**位置**: 多个PHP文件  
**问题描述**: 
- 项目元数据存储在 `data/shortlinks/*.json`
- 速率限制存储在 `data/rate_limit/*.json`
- 筛选条件存储在 `data/filters/saved_filters.json`
- 但项目信息存储在MySQL数据库

**影响**: 
- 数据不一致风险
- JSON文件可能被意外删除
- 备份和恢复复杂

**建议**: 
- 统一使用MySQL数据库存储所有数据
- 或统一使用JSON文件（小规模场景）

**状态**: ⚠️ 待优化

---

#### 问题6: 访问统计未统一数据源
**严重程度**: 🟡 重要  
**位置**: `redirect.php` vs `p/index.php`  
**问题描述**: 
- `redirect.php` 更新JSON文件中的 `visit_count`
- `p/index.php` 更新MySQL数据库中的 `visit_count`
- 两个数据源可能不一致

**影响**: 统计数据不准确  
**建议**: 
- 统一使用数据库记录访问
- 移除JSON文件中的计数逻辑

**状态**: ⚠️ 待修复

---

#### 问题7: 密码保护功能未完全实现
**严重程度**: 🟡 重要  
**位置**: `redirect.php`, `verify_password.php`  
**问题描述**: 
- `verify_password.php` 验证密码并设置session
- 但 `redirect.php` 中未检查密码session
- 数据库中有 `access_password` 字段，但未在发布流程中使用

**影响**: 密码保护功能形同虚设  
**建议**: 
```php
// 在redirect.php中添加密码检查
if (!empty($meta['access_password'])) {
    $sessionKey = 'ce_shortlink_access_' . $slug;
    if (!isset($_SESSION[$sessionKey]) || $_SESSION[$sessionKey] !== true) {
        // 显示密码输入页面
        include 'password.html';
        exit;
    }
}
```

**状态**: ⚠️ 待修复

---

#### 问题8: 速率限制仅基于IP哈希
**严重程度**: 🟡 重要  
**位置**: `deploy_package/publish.php`  
**问题描述**: 
```php
$ipHash = hash('sha256', $ip);
$rateFile = $rateDir . $ipHash . '.json';
```

问题：
1. 同一NAT下的用户共享速率限制
2. 攻击者可以伪造IP绕过限制
3. 使用JSON文件存储，并发访问可能丢失更新

**影响**: 速率限制可能被绕过  
**建议**: 
- 使用Redis或数据库存储速率限制
- 结合API密钥进行限制

**状态**: ⚠️ 待优化

---

#### 问题9: 缺少请求体大小验证
**严重程度**: 🟡 重要  
**位置**: `deploy_package/publish.php`  
**问题描述**: 
```php
$content_length = (int)($_SERVER['CONTENT_LENGTH'] ?? 0);
if ($content_length > $config['max_total_size']) {
    // ...
}
```

问题：
1. `CONTENT_LENGTH` 可以被伪造
2. 应该在读取请求体后验证实际大小
3. 没有验证单个文件大小

**影响**: 可能被大文件攻击  
**建议**: 
```php
// 验证实际接收的数据大小
$totalSize = 0;
foreach ($_FILES['files']['size'] as $size) {
    $totalSize += $size;
    if ($size > 10 * 1024 * 1024) { // 单文件10MB限制
        // 拒绝
    }
}
```

**状态**: ⚠️ 待优化

---

#### 问题10: 数据库未使用事务
**严重程度**: 🟡 重要  
**位置**: `publish.php`, `delete.php`  
**问题描述**: 
- 发布项目时涉及多个数据库操作（插入项目、更新用户、记录活动）
- 但没有使用事务包裹
- 如果中途失败，会导致数据不一致

**影响**: 数据不一致风险  
**建议**: 
```php
try {
    db()->beginTransaction();
    // 插入项目
    // 更新用户
    // 记录活动
    db()->commit();
} catch (Exception $e) {
    db()->rollBack();
    throw $e;
}
```

**状态**: ⚠️ 待优化

#### 问题14: 数据库索引缺失
**严重程度**: 🟢 次要  
**位置**: `database/schema.sql`  
**问题描述**: 
- `projects` 表的 `custom_slug` 字段可能缺少唯一索引

#### 问题15: iOS客户端API密钥存储不安全
**严重程度**: 🟢 次要  
**位置**: `ios/AppConfig.swift`  
**问题描述**: 
```swift
static var apiKey: String {
    Bundle.main.object(forInfoDictionaryKey: "CE_API_KEY") as? String ?? ""
}
```

问题：
- API密钥存储在Info.plist中
- 可以被反编译IPA获取
- HMAC密钥同样不安全

**建议**: 
- 使用服务器端签名代理
- 或使用Keychain存储敏感信息
- 或使用动态令牌机制

## 九、测试路径

### 9.1 核心用户路径

```
1. 首次使用
   └─ 启动App → 引导页 → 创建用户ID → 进入主页

2. 创建项目
   └─ 点击+ → 选择模板/空白 → 编辑代码 → 自动保存
```
