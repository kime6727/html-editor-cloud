# 快速开始指南

## 🚀 5分钟上手 Code Editor – HTML & Preview

### 第一步：打开项目

```bash
# 1. 在Xcode中打开项目
open HTMLPreview.xcodeproj

# 2. 选择目标设备（建议使用真机测试分享功能）
# 3. 点击运行按钮 ▶️
```

### 第二步：创建第一个项目

1. 应用启动后会显示引导页（首次使用）
2. 滑动查看功能介绍
3. 点击"开始使用"进入主界面
4. 系统会自动创建一个默认项目

### 第三步：编辑HTML

```html
<!DOCTYPE html>
<html>
<head>
    <title>我的第一个页面</title>
    <style>
        body {
            font-family: sans-serif;
            padding: 20px;
            background: linear-gradient(135deg, #667eea, #764ba2);
            color: white;
        }
        h1 {
            text-align: center;
        }
    </style>
</head>
<body>
    <h1>Hello World! 🎉</h1>
    <p>这是我的第一个HTML页面</p>
</body>
</html>
```

### 第四步：实时预览

- 在iPhone上：使用底部的视图切换器选择"Split View"
- 在iPad上：左侧编辑器，右侧自动显示预览
- 编辑代码后，预览会自动更新（350ms延迟）

### 第五步：分享预览

#### 方式一：本地网络分享 🆕

1. **点击顶部的QR码图标** 📱
2. **点击"开始分享"按钮**
3. **等待服务器启动**（约1秒）
4. **扫描二维码或复制链接**

```
示例URL: http://192.168.1.100:8080
```

5. **在其他设备上打开链接**
6. **继续编辑，所有设备实时更新** ⚡

#### 方式二：云端发布 🆕

1. **点击"发布"按钮**进入发布中心
2. **选择"云端发布"**
3. **配置发布选项**（可选）：
   - Pro 用户可设置访问密码
   - Pro 用户可设置链接有效期（7天/30天/90天/永不过期）
4. **点击"立即发布"**
5. **等待上传完成**
6. **分享生成的链接或二维码**

```
示例URL: https://html.niceapp.eu.cc/pub/abc12345/index.html
```

#### 方式三：GitHub Pages 🆕

1. **首次使用需要配置GitHub信息**
2. **在发布中心选择"GitHub Pages"**
3. **等待上传完成**
4. **访问生成的GitHub Pages链接**

```
示例URL: https://yourusername.github.io/yourrepo/
```

## 🐛 常见问题

### Q1: 如何开始本地分享？
**A**: 点击顶部工具栏的QR码图标，然后点击"开始分享"。

### Q2: 如何发布到云端？
**A**: 点击"发布"按钮进入发布中心，选择"云端发布"，然后点击"立即发布"。

### Q3: 如何设置访问密码？
**A**: 进入「设置」>「已发布链接」> 选择项目 >「设置密码」（需 Pro）。

### Q4: 免费用户有什么限制？
**A**: 免费用户每月可发布 **3 次**（具体由 `system_config.free_user_monthly_publish_limit` 控制），单次发布后链接 **1 小时** 后过期。升级到 Pro 用户可获得无限发布次数、永久有效期、密码保护等。

### Q5: 如何导出项目？
**A**: 在项目菜单中选择"导出Zip"，或使用发布中心的"导出Zip"快捷操作。

### Q6: 如何管理已发布的链接？
**A**: 进入"设置" > "已发布的链接"，可以查看所有、搜索、复制、分享或停止已发布的链接。

### Q7: 二维码扫描后无法打开？
**A**: 确保两个设备连接到同一WiFi，且已授予本地网络访问权限。

### Q8: 如何配置GitHub Pages？
**A**: 在发布中心选择"GitHub Pages"，首次使用需要输入用户名、仓库名、分支和Access Token（Token 存储于 Keychain）。

## 💡 使用技巧

### 技巧1：使用模板快速开始
不要从零开始，使用预设模板可以节省大量时间。

### 技巧2：访问密码（Pro）
对敏感内容使用 Pro 访问密码，发布后在「已发布链接」中为项目设置密码。

### 技巧3：设置链接有效期
Pro 用户可设置 7天/30天/90天/永不过期；免费用户每次发布固定 1 小时。

### 技巧4：访问统计
发布中心内置访问统计，可查看独立访客、来源、每日趋势等。

### 技巧5：多设备测试
使用本地分享功能，在多个设备上同时测试你的响应式设计。

### 技巧6：GitHub Pages
对于长期展示的作品，使用 GitHub Pages 发布，获得永久链接（Token 安全存储于 Keychain）。
