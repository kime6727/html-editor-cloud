# 分享功能使用指南

## 🎯 功能概述

**Code Editor – HTML & Preview** 提供多种分享方式，让你能够将HTML项目分享给任何人：

1. **本地网络分享** - 通过WiFi分享预览，同一WiFi下的其他设备可以实时查看
2. **云端发布** - 将项目发布到云端，生成永久可访问的链接
3. **GitHub Pages** - 将项目发布到GitHub Pages，适合开发者

## ✨ 核心特性

### 1. 本地网络分享
- 无需上传到云端
- 数据不离开本地网络
- 完全免费，无限制
- 实时同步更新

### 2. 云端发布
- 生成永久可访问的链接
- 自动短链接，方便分享
- 支持自定义短链（如 page.niceapp.eu.cc/p/yourname）
- 可设置链接有效期
- 访问统计和数据分析
- 支持访问密码保护

### 3. GitHub Pages
- 发布到GitHub Pages
- 适合开发者展示作品
- 支持多文件上传
- 免费永久存储

## 📱 使用步骤

### 方式一：本地网络分享

#### 第一步：开始分享

1. 打开你要分享的HTML文档
2. 点击顶部工具栏的 **QR码图标** 🔵
3. 在弹出的界面中点击 **"开始分享"**
4. 等待服务器启动（通常1-2秒）

#### 第二步：连接其他设备

**方式A：扫描二维码（推荐）**
1. 使用其他设备的相机应用
2. 扫描屏幕上显示的二维码
3. 点击弹出的链接通知
4. 浏览器自动打开预览

**方式B：手动输入链接**
1. 复制屏幕上显示的链接（如 `http://192.168.1.100:8080`）
2. 在其他设备的浏览器中粘贴链接
3. 按回车访问

#### 第三步：实时编辑

1. 返回编辑器界面
2. 继续编辑你的HTML代码
3. 所有连接的设备会自动看到更新
4. 无需任何额外操作

#### 第四步：停止分享

1. 返回分享界面
2. 点击 **"停止分享"** 按钮
3. 或直接关闭分享界面
4. 服务器自动停止

### 方式二：云端发布

#### 第一步：打开发布中心

1. 打开你要发布的项目
2. 点击 **"发布"** 按钮进入发布中心

#### 第二步：选择云端发布

1. 在发布中心选择 **"云端发布"**
2. 配置发布选项：
   - **自定义短链**（可选）- 输入你想要的短链后缀
   - **链接有效期**（可选）- 选择7天/30天/90天/永不过期
   - **访问统计**（可选）- 启用访问数据追踪

#### 第三步：发布项目

1. 点击 **"立即发布"**
2. 等待上传完成（通常几秒到几十秒）
3. 查看发布结果：
   - 长链接：完整的访问URL
   - 短链接：自动生成的短链接
   - 自定义链接：你设置的自定义短链（如果有）

#### 第四步：分享链接

1. **复制链接** - 点击复制按钮复制URL
2. **生成二维码** - 点击二维码按钮生成QR码
3. **系统分享** - 使用系统分享功能分享到微信、邮件等
4. **在浏览器中打开** - 直接在Safari中打开预览

### 方式三：GitHub Pages 发布

#### 第一步：配置GitHub

1. 首次使用需要配置GitHub信息
2. 输入以下信息：
   - **用户名** - 你的GitHub用户名
   - **仓库名** - 用于存放项目的仓库
   - **分支** - 通常是 main 或 master
   - **Access Token** - GitHub Personal Access Token

3. 点击 **"验证并保存"**

#### 第二步：发布项目

1. 在发布中心选择 **"GitHub Pages"**
2. 等待上传完成
3. 访问生成的GitHub Pages链接

**解决**：

4. 随时更新作品内容

## 🔒 隐私与安全

### 本地网络分享
- ✅ 代码只在本地WiFi网络传输
- ✅ 不上传到任何云服务器
- ✅ 关闭分享后立即停止服务
- ✅ 其他网络无法访问

### 云端发布
- ✅ HMAC-SHA256签名验证
- ✅ 支持访问密码保护
- ✅ 可设置链接有效期
- ✅ 随时可以停止分享
- ✅ 数据加密传输

### 网络要求
- 📶 本地分享：必须连接到WiFi（不支持蜂窝网络）
- 📶 本地分享：所有设备必须在同一WiFi网络
- 📶 云端发布：需要网络连接
- 📶 某些企业网络可能限制设备间通信

### 权限说明
- 🔐 本地分享需要"本地网络访问"权限
- 🔐 首次使用时系统会弹出授权请求
- 🔐 拒绝授权将无法使用本地分享功能

## ⚙️ 技术细节

### 本地网络分享工作原理
1. 应用在设备上启动HTTP服务器
2. 服务器监听随机端口（如8080）
3. 获取设备的本地IP地址（如192.168.1.100）
4. 生成访问URL和二维码
5. 其他设备通过URL访问HTML内容

### 云端发布工作原理
1. 将项目文件打包上传到云端服务器
2. 服务器生成唯一的访问ID
3. 生成短链接和自定义链接
4. 返回访问URL和二维码
5. 用户通过链接访问云端托管的页面

### 性能特性
- ⚡ 本地服务器启动时间：<2秒
- ⚡ 二维码生成时间：<1秒
- ⚡ 本地更新延迟：<500ms
- ⚡ 云端上传时间：取决于文件大小和网络速度
- ⚡ 支持同时连接：无限制

### 兼容性
- ✅ 支持所有现代浏览器
- ✅ 支持iOS、Android、Windows、Mac
- ✅ 支持HTTP/1.1协议
- ✅ 自动处理CORS跨域

## 🐛 常见问题

### Q1: 二维码扫描后无法打开？
**A**: 检查以下几点：
1. 确保两个设备连接到同一WiFi
2. 确保WiFi不是公共网络（可能限制设备间通信）
3. 尝试手动输入URL而不是扫码
4. 检查是否授予了"本地网络访问"权限

### Q2: 预览不更新？
**A**: 可能的原因：
1. 分享界面已关闭（服务器停止）
2. 网络连接中断
3. 浏览器缓存问题（尝试强制刷新）

### Q3: 无法启动服务器？
**A**: 解决方法：
1. 检查是否授予了本地网络权限
2. 重启应用
3. 检查是否有其他应用占用端口
4. 尝试关闭VPN

### Q4: 云端发布失败？
**A**: 可能的原因：
1. 网络连接问题
2. 文件过大（超过限制）
3. 发布次数已达上限（免费用户）
4. 服务器暂时不可用

### Q5: 自定义短链被占用？
**A**: 解决方案：
1. 尝试其他短链名称
2. 添加数字或字母后缀
3. 使用更长的短链名称

### Q6: 其他网络的设备能访问本地分享吗？
**A**: 不能。这是设计的安全特性：
- 只有同一WiFi下的设备可以访问
- 互联网上的其他人无法访问
- 保护你的代码隐私

### Q7: 可以同时分享多个文档吗？
**A**: 本地分享：
- 一次只能分享一个文档
- 切换文档需要重新开始分享

云端发布：
- 可以发布多个项目
- 每个项目有独立的链接

## 💡 使用技巧

### 技巧1：自定义短链
使用有意义的短链名称，如你的品牌名、项目名等，增强品牌识别度。

### 技巧2：设置有效期
对于临时演示或测试，设置合适的有效期，避免链接长期暴露。

### 技巧3：访问密码
对于敏感内容，设置访问密码，只有知道密码的人才能访问。

### 技巧4：访问统计
启用访问统计，了解有多少人访问了你的链接，优化分享策略。

### 技巧5：批量测试
在多个设备上同时打开预览，一次性测试所有设备的兼容性。

### 技巧6：GitHub Pages
对于长期展示的作品，使用GitHub Pages发布，获得永久链接。
